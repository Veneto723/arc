## Negatives — things I killed, recorded so they are not re-derived

- **N1 — the leaf-pin "gold" is probably unnecessary.** `checkpoint.go:30` (`Parent *CheckpointInfo`) suggested arc should pin the leaf it resumes. But **`--resume` took the chronological tip** in the one revive I could observe (mine, 02:39:50Z; parent `074a0b2b` = the last pre-nap entry, 02:16:46.850Z). The audit's 10,315-token divergence is then explained by **its own finding 5** — *"the originals are not immutable"* — i.e. the conversation moved between resumes, not the resume wandering. **n=1, observed not controlled.** Settling it needs N revives against a frozen conversation (parent landed + each fork's own `created`/`cache_read`), which can ride audit's subject-freeze on the paired rerun.
- **N2 — compaction did NOT demonstrably cause the repeated lesson.** Tempting: compaction #9 (`07-16 05:52:07Z`) sits between `eb53c3c` (02:57:06Z) and `5c4203e` (06:29:52Z) — the same bug written twice. **It does not hold:** the *other* repeat (`1a2e928` 08:21Z → `03e16b0` 12:48Z) has **no compaction in its window**, and `code` **cited `eb53c3c` by hash**, so the lesson was still reachable. 1 of 2, n=1, refuted by its own counter-case. **Dropped.**
- **N3 — arc cannot adopt MAF's compaction.** MAF is Go, in-process, and **owns its executors**, so compaction is a library choice. arc coordinates Claude Code sessions on Windows that it does not own; the conversation *is* the checkpoint and `--resume` *is* the restore. arc's available move is to **observe**, not to compact.
- **N4 — the rest of the repo has nothing for arc.** The workflow graph engine (arc is not one), the providers (arc does not call models), A2A/AG-UI (arc's channel is a file), the checkpoint machinery (N3), 791 KB of generated Azure client. **~2.0 MB of 2.1 MB is inapplicable.** The vein is worked out; the remaining open questions are answered by measuring arc, not by reading more Go.
